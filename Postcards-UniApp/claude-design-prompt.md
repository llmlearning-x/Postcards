# 远方邮政 (Postcards) — 移动端应用原型设计

## 项目概述

「远方邮政」是一款以**邮政明信片**为主题的旅行记录应用。核心概念是：用户每到一个地方，可以创建一张「明信片」，拍下照片、记录位置和心情，并贴上虚拟邮票。应用帮用户将这些明信片按旅程组织，形成一条完整的旅行记忆链。

**平台**：iOS / Android 原生 App 风格
**框架**：移动端小程序/UniApp
**设计目标**：对现有应用进行全面的 UI/UX 重新设计，提升视觉质感、交互体验和品牌一致性。

---

## 核心品牌调性

- **主题**：复古邮政 × 现代旅行
- **情感关键词**：温暖、怀旧、仪式感、探索、记录
- **视觉隐喻**：旧信封、邮戳、手写文字、旅行地图、胶片感
- **设计理念**：每一次旅行，都是一封寄往未来的信

## 全局设计规范

### 圆角
- 大卡片：`24px`
- 小卡片/按钮：`16px` ~ `20px`
- 圆形按钮/头像：`9999px`

### 阴影
- 卡片阴影：`0 4px 16px rgba(0,0,0,0.06)`
- 悬浮按钮：`0 8px 24px rgba(46,125,88,0.3)`

### 间距
- 页面边距：`24px`
- 卡片内边距：`24px` ~ `32px`
- 模块间距：`24px`

### 头部规范
- 所有页面顶部使用**绿色渐变头部**（`#2E7D58` → `#2E6E49`）
- 头部包含标题和副标题，标题使用衬线字体
- 头部下方采用大圆角（`0 0 48px 48px`）过渡到内容区

## 页面清单（共 5 个主 Tab + 3 个子页面）

### Tab 1: 首页 (Home)

**功能**：应用入口，数据概览 + 最近动态

**布局结构**：
1. **绿色渐变头部**：应用名「远方邮政」+ 标语「记录旅途的美好时光」
2. **统计卡片**（4 列横向排列）：
   - 旅程数（邮票图标）
   - 明信片数（照片图标）
   - 收藏数（心形图标）
   - 城市数（定位图标）
   - 每个数据使用大号 Georgia 数字，下方小标签
3. **当前旅程卡片**（如果有进行中的旅程）：
   - 左上角「进行中」状态标签（绿色圆点 + 呼吸动画）
   - 旅程标题 + 目的地
   - 进度条（显示已过天数 / 总天数）
   - 卡片左侧有一条浅绿色边框作为视觉强调
4. **最近明信片列表**（最多 5 条）：
   - 每项包含：缩略图（左）、位置名称 + 城市 + 备注摘要（中）、收藏按钮（右）
   - 列表顶部有「最近明信片」标题 + 「查看全部」链接
5. **空状态**：当没有明信片时，显示占位图标 + 「还没有明信片」+ 引导文案

**交互**：
- 点击明信片项 → 进入详情页
- 点击收藏按钮 → 心形变红 + Toast 提示
- 点击「查看全部」→ 切换到时间轴 Tab

---

### Tab 2: 时间轴 (Timeline)

**功能**：按日期倒序展示所有明信片，形成旅行回忆时间线

**布局结构**：
1. **绿色渐变头部**：标题「时间轴」+ 副标题「回顾旅途的点点滴滴」
2. **时间轴分组列表**：
   - 每组按日期分组（如「2024年5月10日」），组头显示日期 + 当天明信片数量
   - 每组内明信片用**垂直时间线**连接：
     - 左侧：圆点（第一个为绿色高亮 + 光晕，其余为普通绿色）+ 连接线
     - 右侧：横向卡片（缩略图 + 位置名 + 城市 + 备注摘要 + 收藏按钮）
3. **空状态**：大图标配图 + 「还没有明信片」+ 「立即记录」按钮

**交互**：
- 点击卡片 → 进入详情页
- 点击收藏 → 切换收藏状态
- 点击「立即记录」→ 切换到记录 Tab

---

### Tab 3: 记录 (Record) — 核心功能页

**功能**：创建一张新的旅行明信片

**布局结构**（长表单，纵向滚动）：
1. **绿色渐变头部**：「记录明信片」+ 「记录旅途的美好时光」
2. **照片上传区**：
   - 大卡片式上传区域（虚线边框）
   - 默认显示相机图标 + 「点击拍照或选择照片」
   - 选择图片后显示图片预览
3. **表单区域**（每项一个白色卡片）：
   - **位置**：图标 + 输入框 + 定位按钮（点击获取 GPS）
   - **城市**：图标 + 输入框
   - **国家**：图标 + 输入框（默认「中国」）
   - **备注**：图标 + 多行文本框（最大 200 字，带字数统计）
   - **邮票样式**：图标 + 横向排列的 6 种邮票选项（经典/自然/文化/城市/海洋/日落），选中态有绿色边框和背景
4. **实时预览区**：
   - 标题「明信片预览」
   - 模拟真实明信片的预览卡片：顶部绿色条（远方邮政）、照片区、位置信息、备注文字、底部邮票图案 + 「寄往远方」标签
5. **提交按钮**：底部全宽大按钮「寄出明信片」（绿色渐变，圆角胶囊形）

**交互**：
- 点击上传区 → 弹出选择（拍照 / 相册）
- 表单实时更新预览区
- 点击「寄出」→ 保存数据 + Toast「明信片已寄出」+ 1.5 秒后返回首页

---

### Tab 4: 地图 (Map)

**功能**：可视化旅行足迹和统计数据

**布局结构**：
1. **绿色渐变头部**：「旅行地图」+ 「探索去过的地方」
2. **足迹路线卡片**：
   - 标题「我的足迹」+ 城市数量
   - 横向城市路线可视化：一条从左到右的渐变线条，上面有带编号的城市圆点（起点红色，终点绿色，中间绿色），下方显示城市名称
3. **统计网格**（2×2）：
   - 国家数、城市数、明信片数、旅程数
   - 每个是独立小卡片，带图标 + 数字 + 标签
4. **旅程路线列表**：
   - 每条旅程一个卡片：左侧状态图标（进行中🚀/已完成✓/待出发📋）+ 旅程名称 + 目的地 + 日期范围 + 右侧状态标签（不同颜色）

**状态颜色**：
- 进行中：绿色背景标签 + 绿色图标
- 已完成：灰色背景标签 + 灰色图标
- 待出发：蓝色背景标签 + 蓝色图标

### Tab 5: 我的 (Profile)

**功能**：个人信息和设置

**布局结构**：
1. **Hero 区域**（绿色渐变，比普通头部更大）：
   - 顶部居中：「远方邮政」+ 「CHINA POST」小字
   - 中间：大号圆形头像（默认用户图标）

   - 底部：邮筒风格编号卡片（个人邮箱 CN-XXXXXX）
2. **悬浮统计卡片**（与 Hero 区域重叠，白色圆角卡片）：
   - 总旅程 / 明信片 / 收藏（3 列）
3. **设置菜单**（白色卡片分组）：
   - 编辑昵称、重置数据
4. **关于菜单**（白色卡片分组）：
   - 隐私协议、用户协议、关于我们
5. **底部**：版本号 + Slogan「记录旅途，寄往远方」

## 核心交互模式

### 1. 收藏交互
- 点击心形图标 → 状态切换 + 轻微缩放动画 → Toast「收藏已更新」
- 已收藏：红色 `#FF4757`，未收藏：灰色 `#CCCCCC`

### 2. 加载状态
- 全局加载：白色半透明遮罩 + 绿色旋转指示器 + 「加载中...」

### 3. 确认弹窗
- 删除确认：标题「确认删除」+ 内容「确定要删除吗？此操作不可恢复」+ 取消/确认

### 4. 空状态统一规范
- 居中布局：大图标（64px-96px，灰色）+ 主文案（28-32px，深灰）+ 副文案（24-26px，浅灰）+ 可选操作按钮

### 5. 表单校验
- 提交前检查：至少上传照片或填写位置+城市+备注
- 不满足时按钮置灰，点击 Toast「请填写必要信息」

## 设计要求

1. **一致性**：所有页面遵循统一的设计语言（色彩、圆角、阴影、间距）
2. **层次感**：通过卡片阴影、背景色对比建立清晰的信息层级
3. **微交互**：按钮点击有轻微缩放反馈，状态变化有过渡动画
4. **空状态友好**：每个列表/数据区域都有精心设计的空状态
5. **可访问性**：文字与背景对比度充足，触摸目标最小 44px
6. **复古与现代平衡**：邮政主题元素（邮戳、信封、手写感）与现代 iOS 设计规范结合
7. **图片处理**：所有图片使用 `aspectFill` 填充，圆角裁剪
8. **滚动优化**：长列表/表单页面使用 scroll-view，确保流畅滚动
