## 一、产品概述

### 1.1 产品定位

| 属性 | 描述 |
|:---|:---|
| **产品名称** | Postcards |
| **产品类型** | 智能旅行日志工具（移动端 APP + Web 分享页） |
| **核心定位** | 让旅行者"轻松记录、自动整理、美好回忆"的智能旅行日志工具 |
| **Slogan** | Every trip sends you a postcard. / 每一次旅行，都寄给你一张明信片 |
| **目标市场** | 中国市场（主）+ 全球华人市场（次） |
| **目标平台** | iOS 14+、Android 10+、Web H5（分享页） |

### 1.2 核心价值主张

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   🎯 记录零负担          🎯 回忆有温度          🎯 社交有面子  │
│   拍照即记录              AI 自动生成故事         一键分享精美  │
│   3秒完成一条            年度收藏册自动           明信片卡片    │
│                          生成                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 竞争优势

| 竞争优势 | 说明 |
|:---|:---|
| **差异化定位** | "数字时代旅行手账"，情感化设计，与传统相册/游记产品区隔 |
| **AI 能力** | 本地化 AI 故事生成，理解中文旅行语境 |
| **仪式感体验** | 明信片视觉风格、邮戳标记、翻页动效 |
| **轻量化** | 3秒完成记录 vs 传统游记 30 分钟编写 |

## 三、用户故事 (User Stories)

### 3.1 P0 - MVP 核心闭环

#### US-001: 用户注册与首次引导
```
用户故事:
作为新用户，我希望快速完成注册并创建第一条记录，
以便我能立即体验产品的核心价值。

前置条件: 无
触发场景: 首次打开 APP

用户流程:
1. 打开 APP → 显示欢迎页
2. 选择登录方式（手机号/Apple ID/微信）
3. 完成登录 → 进入引导页
4. 授予相册权限 → 进入首页
5. 显示空状态引导 → 点击创建第一条记录
6. 完成记录 → 显示成功反馈

验收标准:
- [ ] 新用户首次打开到完成第一条记录 ≤ 2 分钟
- [ ] 支持手机号/Apple ID/微信三种登录方式
- [ ] 权限申请有明确说明，不强制
- [ ] 引导流程有趣味性，不是枯燥的表单
- [ ] 完成后有成就感反馈（如动画庆祝）
```

#### US-002: 快速拍照记录
```
用户故事:
作为旅行者，我希望拍照后自动记录时间地点，
以便我能在旅行中快速记录，不影响游玩体验。

前置条件: 已登录
触发场景: 首页点击 FAB 或相机入口

用户流程:
1. 点击 FAB → 显示记录方式选择（拍照/语音/文字/相册）
2. 选择拍照 → 打开相机
3. 拍摄照片 → 自动识别时间地点
4. 显示明信片预览 → 可选添加语音/文字
5. 点击保存 → 明信片保存成功，显示邮戳盖章动画
6. 返回首页 → 新的明信片显示在当前旅行卡片中

验收标准:
- [ ] 拍照到显示预览 ≤ 1 秒
- [ ] 地点识别成功率 ≥ 95%（有 GPS 的照片）
- [ ] 无 GPS 照片可手动选择，支持搜索
- [ ] 保存成功显示邮戳盖章动效（400ms）
- [ ] 保存后可撤回（5 秒内显示撤回按钮）
- [ ] 离线状态下可正常拍照，返回首页显示"待同步"标识
```

#### US-003: 时间轴浏览
```
用户故事:
作为旅行者，我希望按时间浏览我的旅行记录，
以便我能回顾每一次旅行的精彩瞬间。

前置条件: 已有明信片记录
触发场景: 底部导航点击"时间轴"

用户流程:
1. 点击时间轴 Tab → 进入时间轴页面
2. 显示旅行筛选器（全部/当前旅行/历史旅行）
3. 明信片按日期分组，瀑布流展示
4. 下拉刷新 / 上拉加载更多
5. 点击明信片 → 进入详情页
6. 详情页支持左右滑动浏览同天其他明信片

验收标准:
- [ ] 时间轴加载时间 ≤ 1 秒
- [ ] 支持按旅行筛选（默认全部）
- [ ] 明信片按日期自动分组
- [ ] 点击明信片进入详情页
- [ ] 详情页支持左右滑动浏览
- [ ] 下拉刷新同步最新数据
- [ ] 上拉加载更早记录（分页加载）
```

#### US-004: 地图足迹展示
```
用户故事:
作为旅行者，我希望在地图上看到我去过的所有地方，
以便我能直观地了解自己的旅行版图。

前置条件: 至少有 1 条明信片记录
触发场景: 底部导航点击"地图"

用户流程:
1. 点击地图 Tab → 进入地图页面
2. 显示全球足迹热力图（去过越多次颜色越深）
3. 每个城市显示邮戳标记
4. 点击邮戳 → 显示该城市的明信片缩略图
5. 可切换视图：热力图 / 单条旅行轨迹 / 3D 地球
6. 底部显示统计：已点亮 N 座城市 / M 个国家

验收标准:
- [ ] 地图加载时间 ≤ 2 秒
- [ ] 支持地图缩放/拖拽操作流畅
- [ ] 邮戳标记点击显示城市明信片
- [ ] 支持旅行轨迹视图（连线动画）
- [ ] 统计数字实时更新
- [ ] 首次到达新城市有"点亮"动效反馈
```

#### US-005: 旅行管理
```
用户故事:
作为旅行者，我希望管理我的旅行（创建/结束/查看历史），
以便我能有序地组织我的旅行记录。

前置条件: 已登录
触发场景: 首页旅行卡片 / 我的页面

用户流程:
1. 首页显示当前旅行卡片（如果有）
2. 点击"开始新旅行" → 输入目的地和开始日期
3. 旅行进行中 → 首页显示旅行进度
4. 连续 3 天无记录 → 推送提示"结束这次旅行？"
5. 点击结束旅行 → 设置结束日期，旅行归档
6. 查看历史旅行 → 我的页面 → 旅行列表

验收标准:
- [ ] 支持创建新旅行（目的地 + 开始日期）
- [ ] 当前旅行显示在首页醒目位置
- [ ] 旅行期间新记录自动归入该旅行
- [ ] 连续 3 天无记录推送提醒
- [ ] 支持手动结束旅行
- [ ] 支持查看/编辑历史旅行
- [ ] 支持删除旅行（需二次确认）
```

### 3.2 P1 - 体验提升

#### US-101: 语音记录
```
用户故事:
作为旅行者，我希望用语音补充记录心情，
以便我能在记录的同时表达情感，而不用打字。

前置条件: 已登录
触发场景: 记录流程中点击语音按钮

用户流程:
1. 在记录流程中点击语音按钮
2. 开始录音 → 显示波形动画
3. 录音完成 → 显示语音时长
4. AI 自动转文字 → 显示转写结果
5. 用户可编辑文字内容
6. 保存后，语音和文字一同保存

验收标准:
- [ ] 支持最长 60 秒录音
- [ ] 录音过程有视觉反馈（波形/时长）
- [ ] 语音转文字准确率 ≥ 90%
- [ ] 转写结果支持手动编辑
- [ ] 支持回放录音
- [ ] 可删除重录
```

#### US-102: 标签系统
```
用户故事:
作为旅行者，我希望给明信片添加标签，

4. 支持添加多个标签

6. 时间轴支持按标签搜索

验收标准:
- [ ] 支持添加多个标签
- [ ] 显示智能推荐（基于地点、历史标签）
- [ ] 支持搜索标签
- [ ] 支持按标签筛选
- [ ] 标签可编辑/删除
- [ ] 推荐算法准确，不推荐违禁词
```

#### US-103: 分享明信片
```
用户故事:
作为旅行者，我希望分享精美的明信片卡片到社交平台，
以便我能与朋友分享我的旅行精彩瞬间。

前置条件: 已有明信片记录
触发场景: 明信片详情页点击分享

用户流程:

2. 点击分享按钮

验收标准:
- [ ] 支持多种分享样式
- [ ] 分享卡片包含品牌水印（可选关闭）
- [ ] 支持一键保存到相册
- [ ] 分享图片清晰度 ≥ 1080p
- [ ] 支持微信/朋友圈/微博/小红书等平台
- [ ] 分享链接可唤起 APP（如已安装）
```

#### US-104: 年度收藏册
```
用户故事:
作为旅行者，我希望每年能自动生成年度旅行收藏册，
以便我能永久保存和分享这一年的旅行回忆。

前置条件: 已有至少 20 条明信片记录
触发场景: 收藏夹页面 / 每年 12 月自动推送

用户流程:
1. 进入收藏夹页面
2. 点击"生成年度收藏册"
3. AI 自动筛选最佳照片
4. 显示收藏册预览（按季节/旅行分组）
5. 用户可替换/调整照片
6. 确认后生成最终版本
7. 可导出为 PDF 或印刷实体册

验收标准:
- [ ] AI 筛选基于照片质量、美学评分、内容多样性
- [ ] 用户可手动替换照片
- [ ] 支持按季节/旅行自动分组
- [ ] 可添加自定义封面/标题
- [ ] 支持导出 PDF（高清版）
- [ ] 收藏册生成过程有动画反馈
```

### 3.3 P2 - 差异化功能

#### US-201: AI 故事生成
```
用户故事:
作为旅行者，我希望 AI 能自动生成旅行故事，
以便我能有高质量的游记内容分享给朋友。

前置条件: Pro 用户，或单次付费
触发场景: 旅行结束后 / 明信片详情页

用户流程:
1. 选择旅行或单条明信片
2. 点击"生成故事"
3. 选择故事风格（轻松/诗意/幽默）
4. 选择语言（中文/英文/日文）
5. AI 生成中 → 显示创意动画
6. 生成完成 → 显示故事内容
7. 用户可编辑/重新生成
8. 保存故事，关联到旅行

验收标准:
- [ ] 生成时间 ≤ 10 秒
- [ ] 支持三种风格：轻松/诗意/幽默
- [ ] 支持中/英/日三种语言
- [ ] 生成内容包含地点串联、情感表达
- [ ] 用户可编辑生成结果
- [ ] 同一旅行支持多次生成不同版本
```

#### US-202: 多人协作旅行
```
用户故事:
作为旅行团成员，我希望与同伴一起协作记录旅行，
以便我们能共同拼凑完整的旅行回忆。

前置条件: Pro 用户
触发场景: 旅行设置页面

用户流程:
1. 创建旅行后，点击"邀请协作者"
2. 生成邀请链接或选择联系人
3. 协作者通过链接加入
4. 协作者可查看/添加自己的记录
5. 记录按贡献者分组显示
6. 旅行结束后统一整理

验收标准:
- [ ] 支持生成邀请链接
- [ ] 支持通过微信/手机号邀请
- [ ] 协作者可查看/添加明信片
- [ ] 记录按贡献者自动分组
- [ ] 支持设置协作者权限（仅添加/可编辑）
- [ ] 旅行主人可移除协作者
```

#### US-203: 3D 地图轨迹
```
用户故事:
作为旅行者，我希望看到 3D 地球上的旅行轨迹，
以便我能以更有趣的方式展示我的旅行足迹。

前置条件: 已有 3+ 条明信片记录
触发场景: 地图页面视图切换

用户流程:
1. 进入地图页面
2. 点击视图切换按钮
3. 选择"3D 地球"视图
4. 地球缓慢旋转，显示所有足迹点
5. 点击足迹点 → 显示该城市照片
6. 可播放旅行轨迹动画

验收标准:
- [ ] 3D 地球加载时间 ≤ 3 秒
- [ ] 支持手势旋转/缩放
- [ ] 足迹点显示正确位置
- [ ] 点击足迹点显示城市信息
- [ ] 轨迹播放流畅无卡顿
- [ ] 轨迹动画时长可调节
```
