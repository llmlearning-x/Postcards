# Postcards - 设计规范与设计语言

## 一、设计哲学

> **"数字时代的旅行手账"**

Postcards 的设计语言融合了**实体明信片的温度**与**数字产品的轻盈**。
我们相信：记录旅行不应该是一件严肃的事，而像收到远方来信一样令人期待。

**核心设计原则**:
1. **温暖而不复古** — 借鉴实体明信片的质感，但保持现代数字体验
2. **记录零负担** — 每一个交互都追求"3 秒内完成"
3. **回忆有仪式感** — 浏览和回顾时，赋予翻阅实体相册的愉悦感
4. **留白即呼吸** — 大量留白让内容成为主角

## 三、字体系统

### 3.1 字体家族

| 用途 | 中文字体 | 西文字体 | 备选 |
|:---|:---|:---|:---|
| **标题/品牌** | 霞鹜文楷 (LXGW WenKai) | Playfair Display | PingFang SC |
| **正文/UI** | 思源黑体 (Noto Sans SC) | Inter | -apple-system |
| **手写/装饰** | 霞鹜文楷手写体 | Caveat | 系统手写体 |
| **数字/数据** | DIN / DIN Alternate | DIN Alternate | SF Mono |

### 3.2 字号规范

| 层级 | 字号 | 字重 | 行高 | 字间距 | 用途 |
|:---|:---|:---|:---|:---|:---|
| **Display** | 32-40px | 700 | 1.2 | -0.02em | Hero 标题、品牌展示 |
| **H1** | 24-28px | 700 | 1.3 | -0.01em | 页面大标题 |
| **H2** | 20-24px | 600 | 1.4 | 0 | 区块标题 |
| **H3** | 16-18px | 600 | 1.5 | 0 | 卡片标题、列表标题 |
| **Body** | 14-16px | 400 | 1.6 | 0.01em | 正文、描述 |
| **Caption** | 12-13px | 400 | 1.5 | 0.02em | 辅助信息、时间戳 |
| **Label** | 11-12px | 500 | 1.4 | 0.03em | 标签、按钮文字 |

### 3.3 排版规则

```
中文正文:
- 每行 25-35 个汉字为最佳阅读宽度
- 段落间距 = 1.5 × 行高
- 避免单字成行 (orphan control)

数字展示:
- 统计数字使用 DIN 字体，加粗
- 大数字每三位加逗号分隔
- 温度、坐标使用等宽字体对齐

邮戳文字:
- 模拟邮戳使用全大写 + 紧凑字间距
- 示例: "TOKYO · 2026.05.01"
```

## 四、图标系统

### 4.1 图标风格

**设计特征**:
- 线框风格 (Outline)，2px 描边
- 圆角端点 (Round cap/join)
- 24×24px 基础画布
- 视觉重量统一，无填充版本为主

**品牌专属图标**:
| 图标 | 描述 | 使用场景 |
|:---|:---|:---|
| 📮 邮筒 | 品牌 Logo 核心元素 | APP 图标、启动页、空状态 |
| 🖼️ 邮票边框 | 照片容器装饰 | 明信片卡片、分享图 |
| 📍 邮戳标记 | 地点标记 | 地图标记、地点信息 |
| ✉️ 信封 | 分享/发送 | 分享按钮、导出功能 |

### 4.2 图标库

使用 **Phosphor Icons** 或 **Feather Icons** 作为基础，
配合 3-5 个品牌定制图标。

## 五、组件规范

### 5.1 明信片卡片 (Postcard Card)

**核心组件，贯穿整个产品**

```
┌─────────────────────────────┐
│                             │
│    ┌───────────────────┐    │
│    │                   │    │

│    │                   │    │     圆角 12px

│                             │
│    📮 TOKYO                 │  ← 邮戳样式地点
│    2026.05.01 · 晴朗 24°C   │  ← 时间天气信息
│                             │
│    "终于来了"               │  ← 用户备注 (可选)
│                             │
│    #日本 #东京 #旅行        │  ← 标签
│                             │
└─────────────────────────────┘

样式:
- 卡片背景: #FFFFFF
- 边框: 1px solid #E8E4DC (模拟纸张边缘)
- 阴影: 0 2px 8px rgba(0,0,0,0.06)

- 内边距: 16px
```

- **紧凑版**: 去掉备注和标签，仅照片+地点，用于时间轴网格
- **精选版**: 增加金色边框 + 星标，用于收藏夹
- **分享版**: 增加底部品牌条 + 二维码，用于社交分享

## 六、动效规范

### 6.1 动画原则

- **有目的**: 每个动效都服务于引导注意力或提供反馈
- **有节制**: 避免过度动画，保持产品轻盈感
- **有温度**: 动效曲线偏柔和，避免机械感

### 6.2 缓动曲线

| 名称 | 曲线 | 用途 |
|:---|:---|:---|
| **柔和进入** | `cubic-bezier(0.16, 1, 0.3, 1)` | 页面进入、卡片出现 |
| **弹性反馈** | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 按钮点击、收藏动画 |
| **自然过渡** | `cubic-bezier(0.4, 0, 0.2, 1)` | 状态切换、页面切换 |
| **线性** | `linear` | 进度条、旋转加载 |

### 6.3 关键动效

**明信片翻页**:
```
时间轴浏览时，左右滑动像翻阅实体明信片
- 过渡: 300ms, 自然过渡曲线
- 效果: 轻微 3D 翻转 (rotateY)
- 阴影: 随翻页角度动态变化
```

**邮戳盖章**:
```
保存明信片时，地点邮戳动画盖下
- 时长: 400ms
- 效果: 从上方落下 + 轻微弹性 + 墨渍扩散
- 音效: 轻微的"盖章"声 (可选)
```

**照片飞入**:
```
从相册导入多张照片时
- 时长: 每张 150ms，错开 50ms
- 效果: 从下方飞入 + 轻微旋转归正
- 终点: 整齐排列成网格
```

**地图点亮**:
```
首次到达新城市，地图标记点亮
- 时长: 600ms
- 效果: 脉冲光环扩散 + 标记从地图浮起
- 反馈: 轻微震动 + 提示 "解锁新城市！"
```

**年度收藏册生成**:
```
AI 生成收藏册时的加载动画
- 时长: 2-3 秒
- 效果: 照片像洗牌一样飞入册子，自动排版
- 完成: 册子合上，显示封面
```

## 七、插画与图形风格

### 7.1 风格定义

**"手绘旅行手账"风格**:
- 线条: 粗细不均的手绘感，模拟钢笔/铅笔
- 色彩: 低饱和度，与主色板一致
- 质感: 轻微纸张纹理叠加
- 留白: 插画内部也保持适度留白

### 7.2 应用场景

| 场景 | 插画内容 | 风格 |
|:---|:---|:---|
| 空状态 | 空邮筒、空白地图 | 简洁线条 |
| 引导页 | 旅行者拍照、翻阅相册 | 场景插画 |
| 成就徽章 | 解锁城市、完成旅行 | 邮票样式 |
| 错误状态 | 迷路、网络中断 | 幽默夸张 |
| 加载状态 | 飞机飞行、火车穿越 | 循环动画 |

### 7.3 邮票边框效果 (CSS 参考)

```css
.postcard-stamp-border {
  /* 模拟邮票齿孔 */
  mask-image: radial-gradient(circle, transparent 4px, black 4.5px);
  mask-size: 12px 12px;
  mask-position: center;

  /* 或使用 SVG 图案 */
  border: 8px solid transparent;
  border-image: url('stamp-edge.svg') 8 round;
}
```

## 八、布局规范

### 8.1 栅格系统

```
移动端 (375px 基准):
┌─────────────────────────────┐
│  16px 边距                  │
│  ┌─────────────────────┐    │
│  │                     │    │  ← 内容区: 343px
│  │                     │    │
│  └─────────────────────┘    │
│  16px 边距                  │
└─────────────────────────────┘

列数: 4 列
列宽: 78px
间距: 12px
安全区: 底部 34px (iPhone X+)
```

### 8.2 间距规范

| Token | 值 | 用途 |
|:---|:---|:---|
| **xs** | 4px | 图标与文字间距、紧凑内联 |
| **sm** | 8px | 列表项间距、标签间距 |
| **md** | 16px | 卡片内边距、区块间距 |
| **lg** | 24px | 大区块分隔、页面边距 |
| **xl** | 32px | 页面区块间距 |
| **2xl** | 48px | 大区块分隔、Hero 间距 |
| **3xl** | 64px | 页面级分隔 |

### 8.3 响应式断点

| 断点 | 宽度 | 布局调整 |
|:---|:---|:---|
| **Mobile** | < 768px | 单列，底部导航 |
| **Tablet** | 768-1024px | 双列时间轴，侧边导航 |
| **Desktop** | > 1024px | 三列，左侧固定导航，右侧详情 |
